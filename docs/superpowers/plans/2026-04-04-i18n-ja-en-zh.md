# KCP サイト 多言語化（ja / en / zh）実装プラン

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** KCP コーポレートサイトを Astro v4 組み込み i18n で日本語・英語・中国語（簡体字）の3言語対応にする。

**Architecture:** ディレクトリ方式（`src/pages/ja/` / `src/pages/en/` / `src/pages/zh/`）。動的 `[lang]` ルートは使わない。翻訳テキストは `src/i18n/[lang].ts` で管理。コンテンツは `src/content/blog/[lang]/` に言語別配置。

**Tech Stack:** Astro v4 built-in i18n、TypeScript strict、既存スタック維持

- [ ] **Step 2: src/i18n/locales.ts を作成**

```typescript
export const LOCALES = ['ja', 'en', 'zh'] as const;
export type Locale = typeof LOCALES[number];
export const DEFAULT_LOCALE: Locale = 'ja';

export const LOCALE_LABELS: Record<Locale, string> = {
  ja: '日本語',
  en: 'English',
  zh: '中文',
};

export function stripLangFromSlug(slug: string): string {
  return slug.replace(/^(ja|en|zh)\//, '');
}

export function getPostsByLang<T extends { slug: string }>(
  posts: T[],
  lang: Locale
): T[] {
  return posts.filter(post => post.slug.startsWith(lang + '/'));
}
```

export async function getStaticPaths() {
  const allPosts = await getCollection('blog');
  const posts = getPostsByLang(allPosts, LANG).filter(p => !p.data.draft);
  return posts.map(post => ({
    params: { slug: stripLangFromSlug(post.slug) },

  }));
}

## 注意事項

- `Astro.currentLocale` は `.astro` フロントマター内でのみ有効。`getStaticPaths` 内では使えない
- `getStaticPaths` 内の言語判定は **LANG定数方式** を使う（ファイルごとに `const LANG = 'en' as const`）
- `src/content/blog/` の slug は移動後 `ja/welcome-to-kcp` の形になる。フィルタ・URL生成は全てこれを前提にする
- 全コンポーネントのハードコードされた href（`/blog/`、`/author/` 等）を `/${lang}/blog/`、`/${lang}/author/` 形式に変更する
- `@astrojs/cloudflare` アダプターは `output: 'static'` では不要。`astro.config.mjs` から削除する
- `wrangler.toml` は変更不要（`[assets] directory = "./dist"` のまま機能する）
